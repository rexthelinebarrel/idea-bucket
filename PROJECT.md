# 灵感桶 (Idea Bucket)

> 一个为创作者打造的随身灵感捕捉 + AI 讨论工具。
> 语音输入降低记录门槛，AI 负责自动连接、追问、展开，让每一个灵感发芽而不是烂掉。

---

## 项目背景

vibecoding 过程中，灵感来得很随机（走路、洗澡、临睡前），当时觉得绝妙，几分钟后忘了。就算记下来了，回头看也不知道怎么展开。更关键的是——没有一个搭档来 bounce ideas，想法就烂在笔记里了。

市面上没有现成方案：
- 备忘录/语音备忘录：能记，但不帮你思考
- ChatGPT 语音对话：是"你去问它"，不是"它在你的灵感库里帮你发现关联"
- Notion/Flomo：能记能整理，但没有 AI 主动帮你连接灵感

灵感桶想做的是：**捕捉 + 连接 + 对话**，缺一不可。

---

## 核心理念

### 灵感桶 vs 笔记软件

笔记软件的核心动作是"记录"。记完就躺在那，等搜索。

灵感桶的核心动作是"放入"。东西扔进去，桶里的灵感会**自动产生关系**——就像一个物理桶，积木丢进去会自然找到接触点，互相支撑。

### 成功标准

> **桶越大，用户越不应该翻。AI 替你翻。**

列表不是检索工具，是 AI 的"推荐流"。用户打开看的是 AI 觉得该看的东西，不是自己去找。

## 产品架构

### 两端职责

| | 手机端 (v1) | 电脑端 (v2) |
|---|---|---|
| 输入 | ✅ 语音投入 | ❌ 不输入 |
| 浏览 | ✅ 列表 + 图谱 | ✅ 图谱为主 |
| 通知 | ✅ 推送 | ❌ |
| 开工 | ❌ | ✅ 查看连接、导出项目 .md |
| 写代码 | ❌ | ❌（交给工作 agent） |

电脑端定位：**只读 + 导出**，不参与灵感投入，不染指代码实现。只做一件事——让用户看清楚灵感之间的关系，产出结构化项目文档，喂给工作 agent。

## 功能详情

### 主界面：极简投入

打开 App，只有一个大按钮。按住说话，松手即走。整个投入过程 ≤ 5 秒。

画面元素：
- 🎤 录音按钮（核心）
- "桶里攒了 X 个点子"（积累感）
- 松手即入桶确认（音频先落盘，不等转写），转写文字稍后异步浮出；弱网/断网下投入也不阻塞，转写失败可事后重试

### 标题自动生成

语音转写后，**当场**从原文提取关键词生成标题，不等 AI 异步处理。

算法逻辑：
- 分词 → 去停用词（"我想""一个""那种"等） → 按词性权重排序（名词 > 动词 > 形容词）
- 取 Top 3-5 个关键词，按原文顺序排列
- 格式：`[MM/DD] 关键词1·关键词2·关键词3`
- 示例：`[03/15] 下载整理·文件分类·自动脚本`

边界处理：
- 语音 < 5 字：直接取原文前 15 字
- 全动词无名词：补最长名词
- 无意义输入：显示 `[MM/DD] 未命名灵感`

### 列表视图（默认浏览）

打开灵感列表 = 准备开工。排序方式：
- **推荐优先**（默认）：AI 发现有连接的、近期活跃的置顶
- 时间倒序
- 按状态筛选

灵感卡片内容：
- 标题（AI 提炼）
- 连接数 + 时间 + 状态标签
- AI 评论摘要
- 关联提示

点进详情：
- 语音转写原文
- 原始音频（保留，可回放）
- AI 处理后的分析/摘要
- 与之相关的灵感列表
- 状态标签（可手动切换）

### 图谱视图（辅助）

类似 Obsidian 关系图谱。节点 = 灵感，连线 = AI 发现的关系。

三种连接类型：

| 类型 | 图谱线型 | 含义 |
|------|---------|------|
| 🧲 合并 | 粗线 | 本质同一件事，建议合并 |
| 🧬 演化 | 虚线箭头 | 同一灵感的多次迭代，指向最新版 |
| 🔀 碰撞 | 实线 | 不同灵感互相启发，组合可能产生新东西 |

节点颜色 = 状态（🌱 灰 / 🔗 蓝 / 🎯 黄 / 🏗️ 绿 / ✅ 暗）

图谱支持手动连接：长按一个节点，再点另一个，弹出"建立连接"。

### AI 后台工作（异步）

灵感投入后，AI 在后台异步处理：
1. 提炼核心要点
2. 自动打标签
3. 扫描桶内是否有相关灵感（合并/演化/碰撞）
4. 发现碰撞 → 进入通知队列
5. 每日定时汇总推送

### 灵感状态流转

```
🌱 原始 ——AI自动——→ 🔗 已连接
                        ↓ 手动 / AI推荐
                     🎯 待开工 ——手动——→ 🏗️ 进行中 ——手动——→ ✅ 完成
```

- 🌱→🔗：AI 自动（发现连接时）
- 🔗→🎯：手动标记，或 AI 推荐后确认
- 🎯→🏗️：手动（开始写了）
- 🏗️→✅：手动（做完了）

### 反悔期

灵感可以删除，反悔期 **30 天**，期间内可在回收站恢复。过期后自动彻底清除（连同音频文件）。灵感只是时机未到。

### 通知推送（待细化）

每天定时（如晚 8 点），AI 汇总当日有价值的连接发现。通知内容有信息量，而非空洞数字：

> "🔗 你的灵感「下载整理」和「桌面清理」可能是同一件事，要看看吗？"

一天最多推送 2-3 条有意义的发现。

### 手动连接

一条灵感的详情页底部有"关联到其他灵感"入口，搜索/浏览后选中建立连接。在图谱视图上也可操作。

## 版本与发布规则（已定）

- **每次发版版本号必须递增**：修复/小调整 → 补丁位（0.3.x），新功能 → 次版本位（0.x.0），破坏性变更 → 主版本位
- **更新机制（0.4.0 起，标准安卓流程）**：App 内检查更新 → 拉取仓库根目录 `release.json`（经 jsDelivr 镜像，国内手机可达）→ 有新版本则应用内下载 APK → 系统安装器弹窗，用户确认后安装。不做 OTA 静默自更新
- **每次出 APK 后必须同步更新 `release.json`**（version / apkUrl / notes）并推送仓库
- 版本号唯一权威来源：`src/version.ts`（不读构建期嵌入的静态值）
- 出包命令：`npx eas build --platform android --profile preview`（EAS 项目 owner：nachdenken）
- （历史教训：曾用 EAS Update OTA + fingerprint runtime，因"版本号参与指纹计算"导致版本递增与 OTA 匹配存在结构性冲突，且用户要求系统安装确认流程，已弃用）

## 更新计划（backlog，按优先级）

1. **前端界面调整**：主界面/列表/详情的视觉与交互打磨（需求收集中）
2. **AI 自动连接（v2 核心，"类 Obsidian"灵感连接）**
   - 架构与供应商解耦（OpenAI 兼容接口，设置页可配）：首选**纯 DeepSeek 轻量路径**——新灵感入桶时把"新原文 + 桶内全部标题"发给 deepseek-chat 直接判定关联（合并/演化/碰撞），几百条规模够用且有 context caching 省钱
   - 桶变大后升级为两段式：embeddings 向量粗筛（硅基流动免费 BGE，或 OpenAI text-embedding-3-small）+ LLM 仅对候选对判定（DeepSeek 无 embeddings 模型，向量环节需别家）
   - 判定进通知队列，用户确认后建立连接（防误连）
   - 成本：轻量路径每条灵感一次 chat 调用（≈免费）；两段式每条一次 embedding + 仅候选对调 LLM
3. **图谱视图**：力导向布局渲染节点+三种连线（合并/演化/碰撞），手机端辅助 tab
4. **导出项目 .md**：框选关联灵感，LLM 综合生成结构化项目文档喂给工作 agent
5. **每日通知推送**：本地定时通知（不依赖推送服务），推送有信息量的连接发现
6. **日志上报升级**：Gitee 镜像仓库，国内用户提 issue 免微信中转
7. **云同步 + 电脑端只读图谱**：v2 后期

## 技术选型（v1 已定）

- 框架：Expo / React Native + TypeScript（SDK 57，expo-router，src/ 目录）
- 录音/回放：expo-audio
- 存储：expo-sqlite（灵感/连接/讨论消息/设置四张表）+ 本地文件系统存音频（document/audio/）
- 语音转写：OpenAI 兼容接口（Whisper 兼容），baseURL / key / 模型名在 App 设置页配置，不接死任何一家服务商
- AI 讨论/展开：OpenAI 兼容 chat completions，同一组配置
- 云同步：v2 再定

## 数据存储

- 本地为主（灵感数据 + 音频文件），v1 无云端依赖
- 云同步（支撑电脑端 v2）

## 版本规划

### v1 — 手机端 MVP
- 极简语音投入（松手即入桶，转写异步补）
- 标题自动生成
- 列表浏览（三种排序 + 搜索 + 状态筛选）
- 灵感详情（原文 + 音频 + AI 摘要 + 关联）
- **AI 讨论：对单条灵感展开/追问/多轮对话**（冷启动第一天就有价值，详见"AI 讨论"一节）
- 手动关联（详情页选择其他灵感建立连接，无类型）
- 状态管理
- 反悔期删除（30 天回收站）
- 本地存储（SQLite + 本地音频文件）

### AI 讨论

立项的初衷是"能讨论灵感的 AI 搭档"，因此 v1 就包含它——而且它不依赖桶里有其他灵感：

- 详情页手动触发"AI 展开"：核心要点 + 追问 + 可展开方向（不自动跑，省 token）
- 围绕这条灵感多轮对话（bounce ideas），对话记录持久保存
- 灵感卡片的摘要：v1 显示原文摘录；AI 自动摘要随 v2 的后台批量分析一起上

### v2 — 电脑端 + 智能连接
- 电脑端图谱只读视图（届时连接数据已积累，图谱才有意义）
- 图谱视图（手机端辅助 tab，含三种连线类型）
- 导出项目 .md（综合关联灵感，生成结构化项目文档）
- AI 自动连接（合并/演化/碰撞）+ 🌱→🔗 状态自动流转
- 每日通知推送（含召回机制）
- 云同步
